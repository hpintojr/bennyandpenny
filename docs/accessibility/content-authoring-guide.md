# Accessible Content Authoring Guide

Use this guide for every new page, case study, image, and document on bennyandpenny.com.

## Headings

- Use one page h1.
- Follow headings in order without skipping levels for visual styling.
- Use CSS classes for appearance rather than changing semantic heading levels.

## Links and buttons

- Use links for navigation and buttons for page-state actions.
- Make link text useful out of context.
- Say where an external link goes and state when it opens a new tab.
- Do not use generic repeated labels such as Click here or Learn more.

## Images

- Write concise alt text for an image that communicates useful information.
- Mark decorative images with empty alt text.
- Do not put essential instructions or headings only inside an image.
- Use real text near logos, signatures, charts, and visual callouts when meaning is important.

## Color and status

- Do not rely on color alone for errors, selected states, availability, or success.
- Include text and, where useful, an icon or border/state cue.
- Check text and control contrast before publication.

## Documents and media

- Add a format label to every download, such as PDF, EPUB, audio, or video.
- Provide accessible source documents with real text and logical reading order.
- Provide captions for video and a text alternative for audio-only content.

## Final content check

- Does the page have a useful title and one h1?
- Are all meaningful images described or intentionally decorative?
- Are links descriptive?
- Can the content be understood without color, hover, animation, or image-only text?
- Does text still fit when enlarged and when spacing is increased?
