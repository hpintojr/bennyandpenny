# Accessibility Preferences QA

Use this checklist for the native Accessibility preferences controller.

- [ ] Trigger is visible in the footer and has the name Accessibility preferences.
- [ ] Trigger works with mouse, Enter, and Space.
- [ ] Dialog opens with focus inside it.
- [ ] Escape closes the dialog and returns focus to the trigger.
- [ ] Close and Done controls return focus to the trigger.
- [ ] Text size buttons announce Standard, Large, and Largest states.
- [ ] Text size does not cause clipping or horizontal scrolling at 320 CSS px.
- [ ] High contrast improves text, border, and focus visibility.
- [ ] Pause animations stops nonessential transition and motion behavior.
- [ ] Readable font swaps decorative type without hiding content.
- [ ] Reset returns all settings to defaults.
- [ ] Preferences persist after a reload.
- [ ] Operating-system reduced-motion preference continues to reduce motion even when the site setting is off.
